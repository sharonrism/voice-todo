# 🎤 语音待办 - Voice Todo

通过语音快速记录待办事项，AI 智能提取，格式简洁清晰，一目了然。

## ✨ 功能特点

- 🎙️ **语音输入**：使用浏览器内置语音识别，支持中文
- 🤖 **AI 智能提取**：Claude API 自动分析口语化内容，提取结构化待办
- 📝 **简洁清晰**：按优先级排序显示，一目了然
- 💾 **本地存储**：数据保存在浏览器中，无需服务器
- 📱 **响应式设计**：支持桌面端和移动端

## 🚀 快速开始

### 1. 准备工作

- 浏览器：**Chrome 88+** 或 **Edge 88+**（必需，其他浏览器不支持语音识别）
- Claude API Key：从 [Anthropic Console](https://console.anthropic.com) 获取

### 2. 使用方法

1. 用浏览器打开 `index.html` 文件
2. 首次使用时，输入您的 Claude API Key
3. 点击麦克风按钮，开始说话
4. 说出您的待办事项，例如：
   - "明天上午10点开会"
   - "周五之前完成项目报告"
   - "给张三打电话，还要买菜"
5. AI 会自动提取待办事项，添加到列表中

### 3. 语音输入示例

**单个任务**：
```
"明天要给客户发邮件"
→ 生成：给客户发邮件（截止：明天）
```

**多个任务**：
```
"今天要买菜做饭，明天开会，后天交报告"
→ 生成 3 个待办事项，带有各自的截止日期
```

**带优先级**：
```
"紧急！今天必须完成季度总结"
→ 生成：完成季度总结（优先级：高，截止：今天）
```

## 🎯 功能说明

### 语音识别

- 使用浏览器内置的 Web Speech API
- 支持中文普通话
- 实时显示识别结果
- 完全免费，无需额外 API

### AI 智能提取

- 使用 Claude 3.5 Sonnet 模型
- 从口语化文本中提取结构化待办
- 自动识别时间信息（今天、明天、下周等）
- 智能判断优先级（高、中、低）

### 待办管理

- **添加**：通过语音或手动编辑
- **完成**：点击 checkbox 标记完成/未完成
- **编辑**：修改待办内容
- **删除**：删除不需要的待办
- **排序**：按优先级自动排序（高→中→低）

### 数据存储

- 使用 localStorage 本地存储
- 数据不会上传到服务器
- 关闭浏览器后数据仍然保留
- API Key 使用 sessionStorage（关闭浏览器后清除，更安全）

## ⚙️ 设置

点击右上角的"⚙️ 设置"按钮，可以：

1. **重新设置 API Key**：更换或修改 API Key
2. **清空所有待办**：删除所有数据（不可恢复）
3. **导出数据**：备份数据为 JSON 文件

## 🔒 隐私与安全

- ✅ 所有待办数据存储在本地浏览器中
- ✅ API Key 仅在浏览器会话期间保存
- ✅ 不会将任何数据发送到除 Claude API 之外的服务器
- ⚠️ 语音识别由浏览器处理，可能会发送到 Google 服务器
- ⚠️ 不建议在公共设备上保存 API Key

## 💡 使用技巧

1. **清晰说话**：在安静环境下，清晰地表达
2. **具体描述**：说出具体的任务，而不是模糊的想法
3. **包含时间**：提到截止时间，AI 会自动提取
4. **多个任务**：可以一次说多个任务，AI 会分别提取
5. **手动编辑**：如果 AI 提取不准确，可以手动编辑修改

## 🐛 常见问题

### 1. 无法使用语音识别

**原因**：浏览器不支持或麦克风权限被拒绝

**解决方法**：
- 使用 Chrome 或 Edge 浏览器
- 检查浏览器设置，允许麦克风访问权限
- 确保麦克风设备正常工作

### 2. API 调用失败

**原因**：API Key 无效或网络问题

**解决方法**：
- 检查 API Key 是否正确
- 确保网络连接正常
- 检查 API 额度是否用完

### 3. AI 提取不准确

**原因**：口语化表达过于复杂或模糊

**解决方法**：
- 更清晰、具体地描述任务
- 使用简单直接的表达
- 手动编辑待办内容

### 4. 数据丢失

**原因**：清理了浏览器缓存或 localStorage

**解决方法**：
- 定期使用"导出数据"功能备份
- 避免清理浏览器数据
